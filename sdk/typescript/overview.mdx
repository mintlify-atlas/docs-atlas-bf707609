The Opik TypeScript SDK provides a comprehensive toolkit for building, tracing, and evaluating LLM applications in TypeScript and JavaScript. Whether you're working with Node.js, Edge runtimes, or modern frameworks, the SDK offers first-class support for monitoring and improving your AI systems.

## Installation

```bash
npm install opik
```

The SDK requires Node.js version 18 or higher and includes TypeScript type definitions.

## Quick Start

<Steps>
  <Step title="Configure your API key">
    Set up your Opik credentials using environment variables:

    ```bash
    export OPIK_API_KEY="your-api-key"
    export OPIK_URL_OVERRIDE="https://www.comet.com/opik/api"
    export OPIK_PROJECT_NAME="your-project-name"
    export OPIK_WORKSPACE="your-workspace-name"
    ```

    Or pass configuration directly to the client:

    ```typescript
    import { Opik } from "opik";

    const client = new Opik({
      apiKey: "your-api-key",
      apiUrl: "https://www.comet.com/opik/api",
      projectName: "your-project-name",
      workspaceName: "your-workspace-name",
    });
    ```
  </Step>

  <Step title="Create your first trace">
    Use the `@track` decorator for automatic instrumentation:

    ```typescript
    import { track, flushAll } from "opik";

    class MyLLMApp {
      @track({ type: "llm" })
      async generateResponse(prompt: string) {
        // Your LLM call here
        return "AI response";
      }

      @track({ name: "process-request" })
      async processRequest(input: string) {
        const response = await this.generateResponse(input);
        return response;
      }
    }

    const app = new MyLLMApp();
    await app.processRequest("Hello, world!");
    await flushAll();
    ```
  </Step>

  <Step title="View your traces">
    Check the Opik dashboard to see your traces, spans, and performance metrics.
  </Step>
</Steps>

## Architecture

The TypeScript SDK is organized into several key modules:

### Core Components

- **OpikClient** (`Opik`): The main client for interacting with the Opik API
- **Trace**: Represents a complete execution flow (e.g., a user request)
- **Span**: Represents individual operations within a trace (e.g., LLM calls, tool invocations)
- **track decorator**: Automatic instrumentation for functions and methods

### Data Management

- **Dataset**: Create and manage test datasets
- **Experiment**: Run evaluations and track results
- **Prompt**: Version control for prompt templates

### Evaluation

- **evaluate()**: Run experiments against datasets with custom tasks
- **evaluatePrompt()**: Evaluate prompt performance with AI SDK integration
- **Metrics**: Built-in heuristic and LLM-judge metrics
